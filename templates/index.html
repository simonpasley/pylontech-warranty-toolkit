Pylontech Battery Health Check

Read every cell, generate a warranty-claim ready report — no battery expertise required.

Read-only diagnostic. Destructive commands (factory / reset / erase / config writes) are blocked at the server.

1. Connect

Plug your USB-RS232 console cable into the master pack's RJ45 console port (NOT the CAN port). Connecting to the master gives you the whole-rack view.

Connecting sends a 1200-baud wakeup frame, then opens the console at 115200 baud. Use the master pack's console port — not CAN, RS485 or Link.

Not connected.

2. Rack overview

Live readings for every pack in the rack. Cell-voltage spread (max cell − min cell) is the at-a-glance health indicator: < 30 mV healthy 30–50 mV degrading > 50 mV failed

A whole-rack scan typically takes ~1 minute for an 8-pack rack (one full diagnostic per pack over RS232). The button stays disabled until it finishes.

4. Event-log dumps (for Pylontech RMA)

Pylontech's RMA team typically asks for "Event data" — the full event log of each suspect pack. The two dumps below are the console-native equivalents of BatteryView's event-log export.

Important — these dumps are pack-local. The log and data event commands only return data from the pack the cable is physically plugged into right now. To dump pack 8's event log: disconnect, move the cable to pack 8's console port, click Connect, then click the dump button. The tool will ask which pack the cable is in before each dump and will stamp that into the filename so you don't mix them up.

The event-log dump usually takes 30–60 seconds (~10 pages). The event-history dump walks every stored item and can take several minutes.

5. Console passthrough (advanced)

Send a Pylontech native console command directly. Only read-only diagnostic commands are accepted — the server enforces an allowlist (info, pwr, bat, soh, stat, log, data, time, alarm, unit, help, ver, sn, fault). Anything that writes config or controls FETs (config …, ctrl …, factory, reset, shut) is rejected with HTTP 403.
